тема переменные и аргументы
переменные хранят значения, а аргументы забируают значения из командной строки
названия переменных могут быть любыми
name="artem"
age=25
считывание происходит через $
echo "$name"
echo "$age"
Есть обычные пользовательские переменные и переменные окружения, которые доступны не 
только в текущем shell, но и в запускаемых из него программах после export.

По соглашению переменные окружения часто пишут в верхнем регистре, 
Аргументы командной строки в Bash доступны через специальные параметры: 
$0 — имя скрипта, $1 — первый аргумент, $2 — второй и так далее, 
а для аргументов после девятого нужно писать ${10}, ${11}.
​
Параметры $@ и $* содержат все аргументы, а 
$# показывает их количество.
например USER или PATH, а пользовательские — в нижнем, чтобы случайно не перезаписать системные значения.
local ограничивает область видимости переменной внутри функции, export делает переменную доступной дочерним процессам, 
а $1, $@, $# и shift помогают разбирать аргументы скрипта.
Аргументы скрипта
$1 — первый аргумент, переданный скрипту в командной строке.

$2, $3 и далее — второй, третий и следующие аргументы.

$# — количество аргументов, переданных скрипту.

Полезные проверки
-f — обычный файл.

-e — объект существует любого типа.

-d — это каталог.

-s — файл существует и не пустой.

! -f — файла нет или это не обычный файл.
-r — есть право на чтение.

-w — есть право на запись.
В конструкции if Bash сначала выполняет команду или проверку в условии, 
и если та возвращает код 0, выполняется 
блок then; если код не 0, Bash переходит к elif или else.

elif проверяется только если предыдущее условие не сработало,
 а else выполняется, когда все предыдущие проверки дали неуспех.
Для Bash полезно запомнить минимум: 0 — успех, не 0 — неуспех; if проверяет именно код возврата команды, 
а exit N завершает скрипт с кодом N
if команда; then ... fi — выполнить блок, если команда завершилась успешно.
else — выполнить запасной вариант, если ничего не подошло.
elif — добавить ещё одну проверку, если предыдущее условие не сработало.
for подходит, когда нужно перебрать список значений, файлов или каталогов, while работает, пока условие истинно, а until выполняется до тех пор, пока условие не станет истинным
for file in *.txt; do
  echo "$file"
done
count=1
while [ "$count" -le 3 ]; do
  echo "$count"
  count=$((count + 1))
done


exit 0 — успешное завершение скрипта.

exit 1, exit 2, exit 10 — завершение с ошибкой или специальным статусом на твой выбор.
